# Data Dictionary

## Table: `city_measurements` (Primary Fact Table)

| Column | Type | Description | Source | Nullable | Example |
|--------|------|-------------|--------|----------|---------|
| `id` | SERIAL | Auto-incrementing primary key | Autogenerated | No | 1 |
| `city` | VARCHAR(100) | City name | CPCB | No | "Delhi" |
| `date` | DATE | Observation date | CPCB | No | 2015-01-01 |
| `pm2_5` | DOUBLE PRECISION | Fine particulate matter <2.5µm (µg/m³) | CPCB | Yes | 85.3 |
| `pm10` | DOUBLE PRECISION | Coarse particulate matter <10µm (µg/m³) | CPCB | Yes | 145.2 |
| `no` | DOUBLE PRECISION | Nitric oxide (µg/m³) | CPCB | Yes | 12.5 |
| `no2` | DOUBLE PRECISION | Nitrogen dioxide (µg/m³) | CPCB | Yes | 35.8 |
| `nox` | DOUBLE PRECISION | Nitrogen oxides (µg/m³) | CPCB | Yes | 48.3 |
| `nh3` | DOUBLE PRECISION | Ammonia (µg/m³) | CPCB | Yes | 22.1 |
| `co` | DOUBLE PRECISION | Carbon monoxide (mg/m³) | CPCB | Yes | 1.45 |
| `so2` | DOUBLE PRECISION | Sulfur dioxide (µg/m³) | CPCB | Yes | 8.7 |
| `o3` | DOUBLE PRECISION | Ground-level ozone (µg/m³) | CPCB | Yes | 65.2 |
| `benzene` | DOUBLE PRECISION | Benzene (µg/m³) | CPCB | Yes | 3.2 |
| `toluene` | DOUBLE PRECISION | Toluene (µg/m³) | CPCB | Yes | 8.5 |
| `xylene` | DOUBLE PRECISION | Xylene (µg/m³) | CPCB | Yes | 1.8 |
| `aqi` | DOUBLE PRECISION | Composite Air Quality Index | CPCB-computed | Yes | 156.0 |
| `aqi_bucket` | VARCHAR(50) | AQI category label | CPCB-computed | Yes | "Moderate" |
| `is_synthetic` | BOOLEAN | Whether this row is synthetically generated | Computed | No | FALSE |
| `data_source` | VARCHAR(100) | Origin of the data | Computed | No | "CPCB" |
| `ingested_at` | TIMESTAMP | When the row was loaded into the database | Computed | No | 2025-07-03 20:15:00 |

**Unique constraint**: `(city, date, data_source)` — one row per city per day per source.

## Table: `city_day` (View — Backward Compatible)

A view over `city_measurements` that renames `pm2_5` to `pm25` for compatibility with legacy code. Includes all provenance columns.

## Table: `stations` (Station Metadata)

| Column | Type | Description | Nullable | Example |
|--------|------|-------------|----------|---------|
| `station_id` | VARCHAR(20) | CPCB station code (PK) | No | "DL001" |
| `station_name` | VARCHAR(200) | Full station name with agency | Yes | "IHBAS, Dilshad Garden, Delhi - DPCC" |
| `city` | VARCHAR(100) | City where station is located | Yes | "Delhi" |

## Enumerations

### AQI Bucket Values

| Bucket | AQI Range | Color | Health Impact |
|--------|-----------|-------|---------------|
| Good | 0–50 | Green | Minimal impact |
| Satisfactory | 51–100 | Light Green | Minor breathing discomfort |
| Moderate | 101–200 | Yellow | Breathing discomfort to sensitive groups |
| Poor | 201–300 | Orange | Breathing discomfort to most |
| Very Poor | 301–400 | Red | Respiratory illness on prolonged exposure |
| Severe | 401–500+ | Dark Red | Healthy people affected; serious impact |

### Data Source Values

| Source | Description |
|--------|-------------|
| `CPCB` | Real data from Central Pollution Control Board (2015-2020) |
| `OpenAQ` | Real data fetched from OpenAQ API |
| `Open-Meteo` | Real data fetched from Open-Meteo API |
| `synthetic` | Programmatically generated synthetic data |
| `synthetic_realistic` | Synthetic data with realistic seasonal patterns |

### Synthetic Flag

| Value | Meaning |
|-------|---------|
| `FALSE` | Real measurement from CPCB, OpenAQ, or Open-Meteo |
| `TRUE` | Generated synthetic data (seasonal + trend + noise) |

## Data Type Conversions (from CSV to Database)

| CSV Column | CSV Type | DB Column | DB Type | Conversion |
|-----------|----------|-----------|---------|------------|
| City | string | city | VARCHAR(100) | As-is |
| Date | string YYYY-MM-DD | date | DATE | Parsed |
| PM2.5 | float | pm2_5 | DOUBLE | As-is |
| AQI_Bucket | string | aqi_bucket | VARCHAR(50) | As-is |
| (all pollutants) | float | (lowercased) | DOUBLE | As-is |

## Notes

- All pollutant measurements are daily averages (24-hour), consistent with CPCB reporting standards
- AQI is computed by CPCB using sub-indices for each pollutant (methodology not publicly documented in detail)
- Synthetic data contains only `aqi` and `pm2_5` columns; all other pollutants are NULL
- Mumbai has significantly lower coverage (~39%) compared to other major cities (>90%)
